Nuevos tipos: Search

Características:

  • Elemento de formulario que permite introducir un término de búsqueda
  • Se permite que se inserte un valor vacío
  • Mismo comportamiento que tipo text, diferente visualización
  • No estandar: uso de atributos results y autosave

Formato:

<input type="search" name="search" />
<input type="search" name="search" results=5 />

Nuevos tipos: Email

  • Se permite que se inserte un valor vacío
  • Si el valor es introducido, debe introducirse un email válido

Nuevos tipos: Color

Características:

  • Elemento de formulario que permite introducir un valor para la representación de un color
  • El valor debe ser una representación de un color simple:
    • longitud de 7 carácteres
    • Caracteres inicial: '#'
    • El resto números hexadecimales en minúscula
  • Puede incluirse un selector de color al hacer foco en el control

Formato:

<input type="color" name="color" />

Nuevos tipos: Number

Características:

  • Elemento de formulario que permite introducir un valor para la representación de un número
  • No se permite introducir el valor vacío
  • Valores numéricos en punto flotante
  • Usado generalmente con los atributos min, max y step

Formato:

<input type="number" name="number" min=1 max=200 />
<input type="number" name="number" min=30 max=300 step=0.5 />

Nuevos tipos: Date

Características:

  • Elemento de formulario que permite introducir un valor para una fecha.
  • Presentación en formato local del cliente.
  • Envío en formato UTC (yyyy-mm-dd).
  • Alternativas:
    • datetime: yyyy-mm-dd HH:MM
    • datetime-local: yyyy-mm-dd HH:MM
    • month: yyyy-mm
    • week: yyyy-mmW
    • time: HH:MM

Formato:

<input type="date" name="date" />

Compatibilidad:

Ejemplo:

Nuevos tipos: Range

Características:

  • Elemento de formulario que permite introducir un valor para la representación de un número en un rango
  • Valores numéricos enteros y punto flotante, no vacío
  • No busca exactitud en el valor, simplifica la interfaz se selección
  • Usado obligatoriamente con los atributos min, max y step
    • min: 0
    • max: 100
    • step: 1
    • value: (min + max) / (max-min)

Formato:

<input type="range" name="range" />

Nuevos Atributos: Pattern

Características

  • Indica un patrón de validación para el valor del campo
  • Se usan expresiones regulares para definir el patrón
  • Se debe incluir un title (y placeholder) para ayudar al usuario a entender el patrón

Formato:

<input type="tel" title="Número de teléfono con 9 digítos, extensión de país optativa" name="phone"
pattern="^(\+\d{2} )?(\d{9})$" />

Nuevos Elementos: Meter

Características

  • Representa una medida dentro de un rango conocido
  • Atributos: min, max, value, high, low y optimum
  • Uso: porcentaje de uso, número de respuestas correctas
  • No sirve para progreso, para ello: progress

Extensibilidad: Pseudo-Selectores

  • :invalid : Activo cuando el valor introducido en el campo NO es válido
    • :invalid { box-shadow: 0px 0px 5px red; }
  • :valid : Activo cuando el valor introducido en el campo es válido
    • :valid { box-shadow: 0px 0px 5px green; }
  • :required : Activo cuando el campo tiene el atributo required
    • :required { box-shadow: 0px 0px 5px #333; }
  • :optional : Activo cuando el campo NO tiene el atributo required
    • :optional { box-shadow: 0px 0px 5px #999; }

Extensibilidad: Javascript

Funciones

  • checkValidity(): comprueba la validez de un elemento o un formulario
  • setCustomValidity(string): establece el mensaje de error

Objetos

  • validity: objecto de comprobación de validación, a su vez tiene:
    • customError: si tiene un customValidity seteado
    • patternMismatch: si no cumple el patron indicado
    • rangeOverflow: si está por encima del máximo valor
    • rangeUnderflow si está por debajo del mínimo valor
    • stepMismatch: si no se encuentra entre los valores válidos para step
    • tooLong: si sobrepasa el valor de maxlenght
    • typeMismatch: si el tipo no es válido
    • valid: si es válido
    • valueMissing: si es requerido y no tiene un valor establecido

Conclusiones

  • Los HTML5 Forms se puede usar desde ya para:
    • Tipos: Email, teléfonos y urls
    • Atributos: Requeridos, patrones, longitud máxima, placeholders, multiple, autofocus
    • Elementos: output
  • El resto:
    • Tipos: como type text
    • Atributos: No considerados
    • Elementos: como divs
  • Otra opción: usar pattern para simular comportamiento en los no soportados
  • Opera es el navegador más compatible con los estándares
